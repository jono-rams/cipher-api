Cipher API

by Jonathan Rampersad

This API provides endpoints for encrypting and decrypting messages using various classical ciphers. Explore the documentation below to learn how to integrate these ciphers into your applications.

Caeser Cipher

This endpoint encrypts or decrypts text using the Caesar cipher. The Caesar cipher is a simple substitution cipher that shifts each letter in the plaintext message a certain number of positions down the alphabet.

POST /api/caeser

Encrypts or decrypts text using the Caesar cipher.

Request Body
{
"text": "string",
"shift": number,
"mode": "encrypt" | "decrypt"
}
Response Body
{
"encryptedText": "string"
}
Examples
JavaScript
const data = {
text: "hello",
shift: 3,
mode: "encrypt"
};

fetch('/api/caeser', {
method: 'POST',
headers: {
'Content-Type': 'application/json'
},
body: JSON.stringify(data)
})
.then(response => response.json())
.then(data => console.log(data));
Python
import requests

data = {
"text": "hello",
"shift": 3,
"mode": "encrypt"
}

response = requests.post('/api/caeser', json=data)
print(response.json())

Vigenere Cipher

This endpoint encrypts or decrypts text using the Vigenere cipher. The Vigenere cipher is a method of encrypting alphabetic text using a polyalphabetic substitution based on the letters of a keyword.

POST /api/vigenere

Encrypts or decrypts text using the Vigenere cipher.

Request Body
{
"text": "string",
"key": "string",
"mode": "encrypt" | "decrypt"
}
Response Body
{
"encryptedText": "string"
}
Examples
JavaScript
const data = {

key: "world",
mode: "encrypt"

fetch('/api/vigenere', {

Atbash Cipher

This endpoint encrypts or decrypts text using the Atbash cipher. The Atbash cipher is a simple substitution cipher that reverses the alphabet, so 'A' becomes 'Z', 'B' becomes 'Y', and so on.

Error Status Codes

The API may return the following error status codes:

Status Code Description Possible Reasons
400 Bad Request The server cannot or will not process the request due to something that is perceived to be a client error (e.g., malformed request syntax, invalid request message framing, or deceptive request routing).
  • Missing required fields in the request body.
  • Invalid values provided for fields (e.g., invalid mode, shift out of range).
  • Incorrect data types used for fields.
500 Internal Server Error The server encountered an unexpected condition that prevented it from fulfilling the request.
  • Unexpected errors in the server-side code.
  • Issues with server resources or dependencies.
